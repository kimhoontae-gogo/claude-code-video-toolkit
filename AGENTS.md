# AGENTS.md

프로젝트는 `AI Agent Native` 개발 방식을 지향한다.

핵심 원칙:

1. 프로젝트 문서는 `docs/**/*.md`로 관리한다.
2. 문서 탐색 시작점은 [docs/README.md](./docs/README.md) 다.
3. 다른 세션의 작업자가 문서만 읽고도 현재 상태와 다음 작업을 이해할 수 있어야 한다.

## 최우선 지침

개발을 하고 나면 항상 `docs/` 문서를 현행화해야 한다.

코드, 구조, 실행 방법, 실험 전략, 마일스톤 상태, 파일 위치가 바뀌면 관련 문서를 즉시 갱신한다.

## 개발 지침

구조, 기능, 실험 흐름, 인터페이스에 영향을 주는 요청은 먼저 계획 문서 작성 또는 기존 계획 문서 갱신을 우선한다.

계획 문서는 사용자와 범위/완료 기준을 합의한 후 구현으로 넘어간다.

## 검증 지침

개발이 완료되면 가능한 검증을 직접 수행한다.

직접 검증이 어려운 경우, 사용자가 재현 가능한 step-by-step 검증 가이드를 제공한다.
